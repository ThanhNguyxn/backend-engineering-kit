## Project Context

This project uses Backend Engineering Kit (BEK) patterns for production-grade code.

## Conventions

### File Structure
- `src/` - Application source code
- `tests/` - Test files
- `.backend-kit/` - BEK patterns and checklists

### Commit Messages
- `feat:` - New features
- `fix:` - Bug fixes
- `refactor:` - Code improvements
- `docs:` - Documentation
- `test:` - Test additions

### Code Style
- Use TypeScript/language-specific strict mode
- Prefer explicit over implicit
- Document public APIs

## Pattern Workflow

When implementing backend features:

1. **Search First**
   ```bash
   bek search "error handling"
   bek search "rate limiting" --scope security
   ```

2. **Apply Pattern**
   - Read Problem section to confirm relevance
   - Follow Solution section exactly
   - Avoid listed Pitfalls

3. **Validate**
   ```bash
   bek gate --checklist checklist-api-review
   ```
