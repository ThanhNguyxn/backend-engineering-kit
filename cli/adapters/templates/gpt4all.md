## System Prompt

```
- Circuit breaker pattern for external dependencies

### Observability
- Structured logging in JSON format
- Correlation IDs for distributed tracing
- Metrics collection (latency, throughput, errors)
- Alerting thresholds and SLO definitions

### Security Best Practices
- Input validation and sanitization
- Authentication (JWT, OAuth2, API keys)
- Authorization (RBAC, ABAC)
- Secret management

## Response Guidelines

1. Provide production-ready code with proper error handling
2. Include logging at appropriate levels (debug, info, warn, error)
3. Add comments explaining non-obvious decisions
4. Suggest unit and integration tests
5. Consider security implications
6. Mention relevant trade-offs
7. Use TypeScript/type hints when applicable
```

## How to Use in GPT4All

### Option 1: Set as Default System Prompt
1. Open GPT4All application
2. Click on Settings (gear icon)
3. Find "System Prompt" or "Default Instructions"
4. Paste the system prompt above
5. Save settings

### Option 2: Per-Chat System Prompt
1. Start a new chat
2. Click the settings icon for that chat
3. Paste the system prompt
4. Begin your conversation

### Option 3: Use LocalDocs (RAG)
1. Place your project files in a LocalDocs folder
2. Enable LocalDocs for your chat
3. GPT4All will use your codebase as context
4. Combine with the system prompt for best results

## Recommended Models for Backend Development

In GPT4All's model browser, look for:
- **Code-focused models** - Best for code generation and review
- **Instruct models** - Good at following specific instructions
- **Larger models (13B+)** - Better quality but need more RAM

Check the "Popular" or "Trending" tabs for community-recommended models.
