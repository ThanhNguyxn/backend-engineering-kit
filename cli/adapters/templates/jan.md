## Setup Instructions

### 1. Create Assistant Directory

```bash
# Linux/macOS
mkdir -p ~/jan/assistants/backend-kit

# Windows
mkdir %USERPROFILE%\jan\assistants\backend-kit
```

### 2. Save Configuration

```bash
# Save the assistant.json file
cp assistant.json ~/jan/assistants/backend-kit/
```

### 3. Restart Jan

After adding the assistant, restart Jan to load the new configuration.

### 4. Select the Assistant

1. Open Jan
2. Click on "Assistants" in the sidebar
3. Select "Backend Engineering Assistant"
4. Start chatting!

## Recommended Models

Jan supports various local models. For backend development, look for:

| Type | RAM Needed | Use Case |
|------|------------|----------|
| 7B models | 8 GB | Quick tasks, general coding |
| 13B models | 16 GB | Complex code, better quality |
| Code-specific | Varies | Best for pure coding tasks |

Browse Jan's Model Hub for the latest recommended models.## Using Jan's API Server

Jan can also serve models via OpenAI-compatible API:

```bash
# Start API server in Jan settings
# Then use with any OpenAI client

curl http://localhost:1337/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "<your-model-id>",
    "messages": [
      {"role": "system", "content": "You are a backend engineering assistant..."},
      {"role": "user", "content": "Implement rate limiting in Express"}
    ]
  }'
```

## Integration with IDE

Use Jan's API with VS Code extensions:

```json

{
  "continue.models": [{
    "title": "Backend Kit (Jan)",
    "provider": "openai",
    "model": "<your-model-id>",
    "apiBase": "http://localhost:1337/v1"
  }]
}
```
